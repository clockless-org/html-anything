html-anything

Turn any file into the best HTML reading experience for that specific content — designed by an LLM.

Not a format converter. The same content type deserves different presentations depending on shape — a 2-person friend chat shouldn't render the same as a 200-person Slack channel. The LLM looks at a sample of the file, designs the right UI for this content, and writes the HTML+CSS+JS. The full data is inlined. Output is one self-contained .html file.

$ npx html-anything chat.zip
→ chat.html  (1 file, opens in any browser)

Live examples

Markdown →

input.md (3 KB, long-form essay) → LLM-designed reading view

The LLM saw the headings + opening + length, picked typography that fits the tone, and decided what navigation this length deserves. Different essay → different design.

WhatsApp export →

_chat.txt (~80 messages between 2 senders) → bubble timeline

The LLM saw "2 senders, 80 messages, 1-month range" and designed a 2-column bubble timeline. Same parser on a 200-person channel would get a different UI — the LLM picks per-content.

CSV →

input.csv (46 rows, 8 columns, 3 numeric) → sortable table + summary

The LLM saw the schema and decided this size deserves a full sortable table. A 50K-row CSV would skew toward charts + virtualization instead.

Meeting transcript — sales call →

input.vtt (43 cues, 4 speakers, pilot-scoping call) → meeting scorecard

The LLM saw a 4-speaker WebVTT transcript with a clear decision (pilot agreed) and a follow-up commitment, and designed talk-time bars, a decisions panel, an action-items table grouped by owner, and a searchable transcript with timestamp chips. A standup with 6 quick speakers would render as a different shape.

Meeting transcript — product review →

input.srt (39 cues, 4 speakers, sprint review) → meeting scorecard

An SRT-format internal review with three decisions, six action items, two open questions, and one risk. The LLM picked the speaker timeline + decisions cards + action-items-by-owner table for this shape — the same parser on a 1:1 sync would produce something tighter.
